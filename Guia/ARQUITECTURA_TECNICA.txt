FLUJO DE DATOS
=======================================================================================

1. CLIENTE (Frontend):
   
   Usuario escribe o pega texto
         ↓
   Click en "Analizar"
         ↓
   JavaScript valida entrada (>3 caracteres)
         ↓
   fetch() POST a http://localhost:5000/analizar
         ↓
   JSON enviado:
   {
       "texto_cifrado": "Uryyb Jbeyq"
   }

2. SERVIDOR (Backend Flask):
   
   app.py recibe petición en /analizar
         ↓
   Extrae texto_cifrado del JSON
         ↓
   Llama a extract_features(texto)
         ↓
   Extrae 17 características estadísticas:
   [
       0: proporción letras
       1: proporción dígitos
       2: proporción especiales
       3: entropía Shannon
       4-15: frecuencias de caracteres
       16: longitud normalizada
   ]
         ↓
   Convierte a array NumPy (1, 17)
         ↓
   MODEL.predict(features)
         ↓
   Obtiene probabilidades para 5 clases

3. RED NEURONAL (Modelo):
   
   Entrada (17 valores)
         ↓
   Dense(64, activation='relu')
         ↓
   Dropout(0.3)
         ↓
   Dense(32, activation='relu')
         ↓
   Dropout(0.3)
         ↓
   Dense(5, activation='softmax')
         ↓
   Salida: [prob_clase0, prob_clase1, prob_clase2, prob_clase3, prob_clase4]
   Ej: [0.02, 0.95, 0.01, 0.01, 0.01] → Clase 1 (Caesar)

4. DESENCRIPTACIÓN:
   
   argmax(predicción) → clase predicha (índice)
         ↓
   Selecciona función de desencriptación:
   - Clase 0: decrypt_plain(texto)
   - Clase 1: decrypt_caesar(texto)
   - Clase 2: decrypt_rot13(texto)
   - Clase 3: decrypt_base64(texto)
   - Clase 4: decrypt_xor(texto)
         ↓
   Aplica función al texto original
         ↓
   Obtiene resultado desencriptado

5. RESPUESTA DEL SERVIDOR:
   
   JSON devuelto a cliente:
   {
       "status": "success",
       "tipo_cifrado": "ROT13",
       "mensaje_descifrado": "Hello World",
       "confianza": 0.9523,
       "texto_original": "Uryyb Jbeyq"
   }

6. CLIENTE ACTUALIZA INTERFAZ:
   
   Recibe JSON
         ↓
   Actualiza elementos del DOM:
   - Tipo cifrado: span#tipoCifrado
   - Confianza: barra visual
   - Mensaje: p#mensajeDescifrado
   - Original: p#textoOriginal
         ↓
   Muestra sección de resultados
         ↓
   Usuario ve el resultado
         ↓
   Puede copiar o hacer nuevo análisis

CARACTERÍSTICAS DEL MODELO
=======================================================================================

ENTRADA (17 características):

Seleccionadas porque son:
✓ Invariantes al cambio de caso
✓ Robustas a variaciones pequeñas
✓ Diferenciadoras entre cifrados
✓ Eficientes de calcular

Cálculo:

1. Proporción de letras
   letters_count / total_chars

2. Proporción de dígitos
   digits_count / total_chars

3. Proporción de especiales
   special_count / total_chars

4. Entropía Shannon
   Σ(-p * log2(p)) / 8, donde p = freq/total
   
   Interpreta la aleatoriedad:
   - Texto plano: baja (~3-4)
   - Base64: alta (~5-6)
   - XOR: muy alta (~7-8)

5-16. Frecuencias top 12 caracteres
   
   Se cuentan los 12 caracteres más frecuentes
   y se calcula: count / total_chars para cada uno

17. Longitud normalizada
   
   min(len(texto) / 1000, 1.0)
   
   Ayuda a distinguir:
   - Textos muy cortos (difícil clasificar)
   - Textos normales (mejor predicción)

CLASES DE SALIDA:

0 → "Texto plano"
   - Distribución de caracteres normal
   - Confianza: muy alta con buen texto
   - Ejemplo: "Hello World"

1 → "Caesar"
   - ROT variable (no siempre 13)
   - Entropía media-baja
   - Confianza: alta con muchas letras
   - Ejemplo: "Khoor Zruog" (shift=3)

2 → "ROT13"
   - Caso especial de Caesar (shift=13)
   - Distribución característica
   - Confianza: muy alta
   - Ejemplo: "Uryyb Jbeyq"

3 → "Base64"
   - Caracteres: A-Za-z0-9+/=
   - Proporción de = al final
   - Entropía media-alta
   - Confianza: alta
   - Ejemplo: "SGVsbG8gV29ybGQ="

4 → "XOR"
   - Muchos caracteres especiales
   - Entropía muy alta
   - Distribución uniforme
   - Confianza: media (ambiguo)
   - Ejemplo: Caracteres binarios aleatorios

ARQUITECTURA MLP:

```
Entrada
  ↓ (17 features)
Layer 1: Dense(64)
  ↓
Activation: ReLU
  ↓
Dropout(0.3) → Previene overfitting
  ↓
Layer 2: Dense(32)
  ↓
Activation: ReLU
  ↓
Dropout(0.3) → Previene overfitting
  ↓
Layer 3: Dense(5)
  ↓
Activation: Softmax → Probabilidades [0,1]
  ↓
Salida → [p0, p1, p2, p3, p4]
```

ENTRENAMIENTO:

- Datos: 1,250 muestras (250 × 5 clases)
- Split: 80% train, 20% validation
- Épocas: 50
- Batch Size: 32
- Optimizer: Adam (lr=0.001)
- Loss: Sparse Categorical Crossentropy
- Precisión final: ~90-95%

PREDICCIÓN:

- Tiempo: ~50-100ms por análisis
- Batch size: 1 (en tiempo real)
- Output: 5 probabilidades
- Argmax: obtiene clase predicha
- Confianza: max(probabilidades)


ALGORITMOS DE DESENCRIPTACIÓN
=======================================================================================

TEXTO PLANO:
   def decrypt_plain(text):
       return text
   
   Lógica: Devuelve igual

CAESAR:
   def decrypt_caesar(text):
       for shift in range(26):
           decrypted = rotate_letters(text, -shift)
           if count_common_words(decrypted) > best:
               best = decrypted
       return best
   
   Lógica: Fuerza bruta 26 rotaciones
   Selecciona basado en palabras comunes inglesas

ROT13:
   def decrypt_rot13(text):
       return decrypt_caesar(text)  # shift=13
   
   Lógica: Especial de Caesar con shift=13

BASE64:
   def decrypt_base64(text):
       return base64.b64decode(text).decode('utf-8')
   
   Lógica: Decodificación estándar
   Manejo de errores: ignora bytes inválidos

XOR:
   def decrypt_xor(text, key=42):
       return ''.join([chr(ord(char) ^ key) for char in text])
   
   Lógica: XOR bit a bit con clave
   Default: clave=42


SEGURIDAD Y LIMITACIONES
=======================================================================================

PUNTOS FUERTES:

✓ Procesamiento local (no hay servidores externos)
✓ Sin almacén de datos (todo en memoria)
✓ Sin tracking o logging de usuarios
✓ CORS bien configurado

LIMITACIONES:

✗ No detecta:
  - Cifrado simétrico (AES, DES)
  - Cifrado asimétrico (RSA)
  - Compresión (ZIP, GZIP)
  - Hashes (SHA, MD5)

✗ Requiere:
  - Texto de suficiente longitud
  - Caracteres legibles para algunos cifrados
  - Parámetros conocidos (XOR key)

✗ Precisión afectada por:
  - Textos muy cortos (<20 caracteres)
  - Idiomas no-inglés
  - Variaciones del algoritmo
  - Datos de entrenamiento limitados
